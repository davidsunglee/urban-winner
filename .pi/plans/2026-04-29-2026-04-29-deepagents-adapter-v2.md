## Goal

Replace the v1 stub at `frameworks/deepagents/` with a working DeepAgents adapter that satisfies `shared/contract.md` and `shared/task-spec.md`. The adapter reads one JSON request envelope from stdin, runs a DeepAgents loop (model: `claude-sonnet-4-6` via Anthropic) against the per-cell worktree at `input.repo_path`, edits the worktree in place to fix the failing test, and writes one contract-compliant JSON response envelope to stdout. Logs go to stderr; the adapter never mutates `.git/` or the harness-owned case venv. On success the harness end-to-end produces `runs/CURRENT/deepagents/py-parse-duration-001/` with `meta.status == "ok"`, `scoring.schema_validity == true`, and `visible_test_outcome == hidden_test_outcome == "pass"`.

## Architecture summary

The harness invokes `frameworks/deepagents/run.sh` with `cwd=frameworks/deepagents/`. `run.sh` shells out to `uv run python adapter.py`, which is the single Python entry script implementing the adapter. The adapter:

1. **Parses request envelope** from stdin (`task_id`, `input.*`, `config.*`).
2. **Builds the agent** via `create_deep_agent(model=init_chat_model("anthropic:<model>"), backend=<backend>, tools=[submit_report], system_prompt=<bugfix-task-prompt>)`. The chosen approach (per spec `## Approach`) is `LocalShellBackend` rooted at `input.repo_path`; the spec mandates a fallback to `FilesystemBackend(root_dir=input.repo_path)` plus a custom `@tool def shell(command)` tool when `LocalShellBackend.__init__` does not expose a working-directory knob (or when its filesystem tools are not rooted by it). The implementer picks one path at implementation time by inspecting the installed `deepagents` source and bakes it in (not runtime-conditional).
3. **Invokes the agent** with one human message containing `failing_test_command`, `failure_output`, and `edit_constraints`. LangGraph `recursion_limit` is set to `2 * max_steps` (multiplier per spec Open Questions; counts graph steps not tool calls). A SIGALRM-based soft deadline derived from `config.timeout_s` wraps the invocation so the adapter emits its own timeout envelope before the harness hard-kills the subprocess.
4. **Watches the message stream** for a `submit_report` tool call. The tool's Pydantic model exactly matches `shared/task-spec.md`'s `output` schema (`root_cause`, `summary`, `changed_files`, `tests_run[]`, `evidence`, `confidence`). Last-call-wins if the agent invokes it more than once.
5. **Converts message history** to contract trace (`steps[]`, `tokens.{input,output}`, `latency_ms`) by summing `usage_metadata.input_tokens`/`output_tokens` from `AIMessage`s and emitting one `step` per `AIMessage`/`ToolMessage`.
6. **Emits the final envelope** to stdout: `{task_id, output, trace, error: null}` and exits 0. Any failure (model error, recursion-limit hit, soft-deadline reached, no `submit_report` observed, parse error) is caught at the top level: the adapter writes a contract-compliant envelope with `error.message` populated and exits non-zero. Stdout is **only** the final envelope — DeepAgents/LangChain logging is redirected to stderr at startup.

`frameworks/deepagents/` ships its own `pyproject.toml` and `uv.lock`; the manifest declares `setup: "uv sync --frozen"`. The harness-owned case venv (on `PATH`) is for test commands the agent runs; the deepagents adapter runs from its own `uv` environment.

- **Python ≥ 3.11** (matches the harness; deepagents requires Python 3.11+).
- **deepagents** (the agent loop).
- **langchain** + **langchain-anthropic** (model provider; `init_chat_model("anthropic:...")` requires `langchain-anthropic`).

  ```python
  def _read_request() -> dict[str, Any]:
      raw = sys.stdin.read()
      if not raw.strip():
          raise RuntimeError("empty stdin: expected one JSON request envelope")
      try:
          obj = json.loads(raw)
      except json.JSONDecodeError as exc:
          raise RuntimeError(f"stdin is not valid JSON: {exc}") from exc
      if not isinstance(obj, dict):
          raise RuntimeError("request envelope must be a JSON object")
      for key in ("task_id", "input", "config"):
          if key not in obj:
              raise RuntimeError(f"request envelope missing required key: {key}")
      inp = obj["input"]
      for key in ("repo_path", "failing_test_command", "failure_output", "edit_constraints"):
          if key not in inp:
              raise RuntimeError(f"input missing required key: {key}")
      cfg = obj["config"]
      for key in ("model", "max_steps", "timeout_s"):
          if key not in cfg:
              raise RuntimeError(f"config missing required key: {key}")
      return obj
  ```

- [ ] **Step 8: Convert message history to trace** — add:
  ```python
  def _messages_to_trace(messages: list, latency_ms: int) -> dict[str, Any]:
      steps: list[dict[str, Any]] = []
      input_tokens = 0
      output_tokens = 0
      saw_usage = False
      for msg in messages:
          if isinstance(msg, AIMessage):
              usage = getattr(msg, "usage_metadata", None) or {}
              if usage:
                  saw_usage = True
                  input_tokens += int(usage.get("input_tokens", 0))
                  output_tokens += int(usage.get("output_tokens", 0))
              tool_calls = getattr(msg, "tool_calls", None) or []
              if tool_calls:
                  for tc in tool_calls:
                      steps.append({
                          "kind": "tool_call",
                          "name": tc.get("name", ""),
                          "args": tc.get("args", {}),
                          "result": {},
                      })
              else:
                  steps.append({
                      "kind": "model_call",
                      "name": "ai_message",
                      "args": {},
                      "result": {"content": _stringify_content(msg.content)},
                  })
          elif isinstance(msg, ToolMessage):
              if steps and steps[-1]["kind"] == "tool_call" and not steps[-1]["result"]:
                  steps[-1]["result"] = {"content": _stringify_content(msg.content)}
              else:
                  steps.append({
                      "kind": "tool_call",
                      "name": getattr(msg, "name", "") or "",
                      "args": {},
                      "result": {"content": _stringify_content(msg.content)},
                  })
      trace: dict[str, Any] = {"steps": steps, "latency_ms": int(latency_ms)}
      if saw_usage:
          trace["tokens"] = {"input": int(input_tokens), "output": int(output_tokens)}
      else:
          trace["tokens"] = {"input": 0, "output": 0}
      return trace

  def _stringify_content(content: Any) -> str:
      if isinstance(content, str):
          return content
      if isinstance(content, list):
          parts: list[str] = []
          for item in content:
              if isinstance(item, dict) and "text" in item:
                  parts.append(str(item["text"]))
              else:
                  parts.append(str(item))
          return "".join(parts)
      return str(content)
  ```

  ```python
          timeout_s = float(cfg["timeout_s"])
          repo_path = input_obj["repo_path"]

          if not os.path.isdir(repo_path):
              raise RuntimeError(f"input.repo_path does not exist or is not a directory: {repo_path}")

          # Soft deadline: emit our own contract-valid envelope before the harness hard-kills us.
          # Reserve 5s of headroom for envelope serialization + stdout flush. Floor at 5s so a
          # tiny config.timeout_s still produces a sensible (if immediate) deadline rather than
          # a negative timer.
          soft_deadline_s = max(5.0, timeout_s - 5.0)
          _DEADLINE["deadline"] = t0 + soft_deadline_s

          agent = _build_agent(model_name=model_name, repo_path=repo_path)
          user_text = _user_message(input_obj)
          recursion_limit = max(10, max_steps * 2)

          # SIGALRM-based soft deadline. Works in the main thread (Python raises the exception
          # between bytecode instructions when control returns from blocking syscalls).
          signal.signal(signal.SIGALRM, _on_timeout)
          signal.setitimer(signal.ITIMER_REAL, soft_deadline_s)
          try:
              result = agent.invoke(
                  {"messages": [{"role": "user", "content": user_text}]},
                  config={"recursion_limit": recursion_limit},
              )
          finally:
              signal.setitimer(signal.ITIMER_REAL, 0)  # disarm even on error

          messages = result.get("messages", []) if isinstance(result, dict) else []
          latency_ms = int((time.monotonic() - t0) * 1000)
          trace = _messages_to_trace(messages, latency_ms)

          if not _CAPTURED_REPORT:
              envelope = {
                  "task_id": task_id,
                  "output": None,
                  "trace": trace,
                  "error": {"message": "agent did not call submit_report"},
              }
              _emit_envelope(envelope)
              return 1

          envelope = {
              "task_id": task_id,
              "output": dict(_CAPTURED_REPORT),
              "trace": trace,
              "error": None,
          }
          _emit_envelope(envelope)
          return 0
      except BaseException as exc:
          # Disarm the timer in case the exception was not _AdapterTimeout itself.
          try:
              signal.setitimer(signal.ITIMER_REAL, 0)
          except Exception:
              pass
          latency_ms = int((time.monotonic() - t0) * 1000)
          tb = traceback.format_exc()
          sys.stderr.write(tb)
          sys.stderr.flush()
          envelope = {
              "task_id": task_id,
              "output": None,
              "trace": {"steps": [], "tokens": {"input": 0, "output": 0}, "latency_ms": latency_ms},
              "error": {"message": f"{type(exc).__name__}: {exc}"},
          }
          try:
              _emit_envelope(envelope)
          except Exception:
              pass
          return 1

  if __name__ == "__main__":
      sys.exit(main())
  ```
  Notes:
  - `BaseException` is intentional: it catches `KeyboardInterrupt`, `SystemExit`, `_AdapterTimeout`, and `GraphRecursionError` (LangGraph throws this when `recursion_limit` is hit). Any failure produces a contract-compliant envelope; the harness then sees `error_reason="nonzero_exit"` and `schema_validity=true` (envelope is well-formed; `output` is null which the harness records as agent-output-invalid but envelope-valid).
  - `task_id="unknown"` is used if the failure occurs before stdin parsing.
  - `signal.SIGALRM` is POSIX-only and fires in the main thread, which is where `agent.invoke` runs (it is synchronous). On macOS/Linux the harness already runs the adapter under a Unix shell, so SIGALRM is available. If a tool call is blocked in a long subprocess `wait()`, the signal is queued and the exception fires when control returns to Python — for hard-bounded shell timeouts inside Approach B, the shell tool also derives its own per-call timeout from `_DEADLINE` (see Task 3-alt Step 5).

  ```python
  _REPO_PATH: dict[str, str] = {}  # set in main() before agent invoke
  _SHELL_TIMEOUT_S = 60  # per-call ceiling; effective timeout is min(this, remaining adapter deadline - 1s)
  _SHELL_OUTPUT_CAP_BYTES = 1024 * 1024  # 1 MiB per call, matches deepagents defaults
  _SHELL_MIN_TIMEOUT_S = 2.0  # bail out rather than starting a command that has < 2s to run

  @tool("shell")
  def shell(command: str) -> str:
      """Run a shell command in the repository root and return its stdout+stderr (capped). Use for pytest, git diff, ls, etc."""
      cwd = _REPO_PATH.get("repo_path")
      if not cwd:
          return "shell error: repo_path not set"
      timeout = float(_SHELL_TIMEOUT_S)
      deadline = _DEADLINE.get("deadline")
      if deadline is not None:
          remaining = deadline - time.monotonic()
          if remaining <= _SHELL_MIN_TIMEOUT_S:
              return (
                  f"shell error: only {remaining:.1f}s left before adapter deadline; "
                  "refusing to start command. Call submit_report now with what you have."
              )
          # Cap below the adapter deadline (1s slack so the subprocess can be reaped before SIGALRM fires).
          timeout = max(_SHELL_MIN_TIMEOUT_S, min(_SHELL_TIMEOUT_S, remaining - 1.0))
      try:
          completed = subprocess.run(
              ["/bin/sh", "-c", command],
              cwd=cwd,
              env=os.environ.copy(),
              capture_output=True,
              timeout=timeout,
          )
      except subprocess.TimeoutExpired:
          return f"shell error: command timed out after {timeout:.1f}s"
      out = (completed.stdout or b"") + (completed.stderr or b"")
      truncated = len(out) > _SHELL_OUTPUT_CAP_BYTES
      out = out[:_SHELL_OUTPUT_CAP_BYTES]
      tail = "\n[output truncated]" if truncated else ""
      return f"exit_code: {completed.returncode}\n{out.decode('utf-8', errors='replace')}{tail}"
  ```

## Risk Assessment

- **`LocalShellBackend` API drift.** The deepagents library is pre-1.0 and has actively churned (`backends`, `middleware`, etc.). The constructor parameter for cwd-pinning may be `root_dir`, `cwd`, or absent; even if present, the filesystem methods may resolve paths from the process CWD instead of that parameter. Mitigation: Task 2 inspects the installed source (constructor signature, `execute` cwd plumbing, AND every filesystem method's path resolution) before writing `adapter.py`, and the Approach B fallback is fully specified. The fallback adds ~25 lines of code and does not change the public envelope.
- **Anthropic API non-determinism.** The acceptance criterion `visible_test_outcome == "pass"` depends on the model fixing the bug. The bug (`UNITS = {"m": 60}` missing `s` and `h`) is small and well-scoped, and the spec calls out that single-shot runs may be model-flaky. Mitigation: the README and acceptance criteria explicitly note this; if Task 7 Step 6 fails on a single run, re-run the cell (`just eval deepagents py-parse-duration-001`) before debugging the adapter. The adapter must not be the source of flakiness — i.e., the agent must always be allowed to fail naturally rather than being short-circuited or retried by the adapter.
- **`recursion_limit` undercount.** LangGraph's `recursion_limit` counts graph steps rather than tool calls. The `2 * max_steps` multiplier is a heuristic. If a run hits the limit, LangGraph raises `GraphRecursionError`, which our `except BaseException` catches and converts to a contract-valid error envelope. Tuning the multiplier upward (e.g., to 3x) is a follow-on adjustment and does not require an adapter rewrite.
- **`config.timeout_s` and SIGALRM.** Python's `signal.SIGALRM` only fires in the main thread and is delivered between bytecode instructions. A long blocking call inside a C extension (e.g., `subprocess.run` with no `timeout=`) may delay the exception. Mitigation: the adapter sets a soft deadline `max(5.0, timeout_s - 5.0)` (5 s headroom for envelope emission) so even with delivery delay we generally beat the harness's hard-kill. Approach B's shell tool further caps `subprocess.run(timeout=...)` to the remaining deadline so the longest-running tool calls return control to Python well before SIGALRM fires. Approach A relies on `LocalShellBackend.execute`'s built-in timeout (deepagents' default), which is bounded; if that proves too long in practice, the mitigation is to reduce the per-call timeout in the deepagents config or switch to Approach B.
- **`ANTHROPIC_API_KEY` not in `agent_env`.** `build_agent_env` only forwards keys the manifest lists in `env`. If the manifest's `env` field is wrong, the adapter sees no key and fails. Task 4 explicitly sets `env: ["ANTHROPIC_API_KEY"]`; Task 7 Step 1 verifies the key is in the parent env.
- **Stdout contamination.** DeepAgents' middleware or third-party tracing may write to stdout. Mitigation: Task 3 Step 2 redirects `logging` to stderr at startup. The smoke test in Task 5 Step 3 confirms the error path emits exactly one JSON line on stdout. If a future DeepAgents version writes to stdout outside the logging module, the fix is to dup-redirect file descriptor 1 to stderr at startup, then re-emit only via `_emit_envelope` (a one-line change to `main()` — `os.dup2(2, 1)` followed by reopening a Python `sys.stdout` handle pointing at the original FD captured before the dup).
- **Venv mutation.** The agent has shell access. The system prompt forbids `pip install`/`uv sync`/`uv add`, but a determined agent could bypass the prompt. The harness fingerprints the case venv before/after each cell and reports `venv_mutated: true` if it detects a change. Mitigation: the prompt's hard-constraints section is explicit; if `venv_mutated` is observed in production, tighten the prompt rather than blocking shell commands (which would limit legitimate `pytest` runs).
- **`.git/` mutation.** Same threat model. The harness derives the diff from a temporary index against pristine HEAD (`evals/evals/pipeline.py:81-126`), so even if the agent commits inside the worktree, the harness still computes the canonical diff against the original state. The system prompt is the primary defense; the diff derivation is the safety net.
